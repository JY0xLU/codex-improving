# Inbox

Use this folder only as a short-lived quarantine buffer for inferred, ambiguous, competing, or otherwise unresolved signal.

This is not a public third layer and it is not where explicit strong signal should live.

## Put Signal Here Only When

- it is inferred rather than directly stated
- it is still too weak to guide future behavior
- it is still competing with another route
- it may become useful later, but is not yet ready for `ACTIVE.md` or `LEARNINGS.md`

## Do Not Put Signal Here When

- the user gave an explicit correction with a clear destination
- the user gave a durable preference with clear cross-task reuse
- the route is already clearly hot and should affect behavior now
- the route has already clearly won and should become reusable path memory

## Expected Lifecycle

- unresolved signal may stay here briefly
- strong signal should move out quickly
- stale or losing signal should be archived, not left to rot

## Guardrail

Do not rewrite `ACTIVE.md` or `LEARNINGS.md` from inbox blindly. Promotion should still preserve source trace and auditability.
